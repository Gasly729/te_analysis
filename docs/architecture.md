# Architecture (TL;DR)

This repo is a CCDS wrapper. It does **not** reimplement any upstream logic.

- Upstream execution: `vendor/snakescale` (submodule, zero edits)
- Downstream execution: `vendor/TE_model` (submodule, zero edits)
- Our code: `metadata.csv → project.yaml + FASTQ symlinks`, that's it
- Entry: `make all STUDY=<GSE>`

Full architecture: see `te_analysis_top_level_design_v1.md`.

## T4 data flow (as of Commit K1)

```text
data/raw/metadata.csv            (run-level, 30 cols; H2 + J1 enriched)
data/raw/{organism}/{GSE}/*.fastq.gz   (real FASTQ, 4689 files)
        │
        │  src/te_analysis/stage_inputs.py
        │    --metadata ... --study <GSE> --out <out>
        ▼
<out>/project.yaml                          (snakescale L2, template-overridden)
<out>/staged_fastq/{GSE}/{GSM}/{SRR}_[12].fastq.gz   (symlinks -> data/raw/...)
        │
        │  (T5) src/te_analysis/run_upstream.py — not yet implemented
        ▼
<out>/ribo/ ... .ribo                        (RiboFlow output)
        │
        │  (T6) src/te_analysis/run_downstream.py — not yet implemented
        ▼
data/processed/te/<GSE>/*.{rda,csv}          (TE_model output)
```

Design invariants:

- `vendor/` is read-only (GC-3 SSOT; submodule SHA pinned in `.gitmodules`)
- `data/raw/metadata.csv` is the sole experiment registry (no sidecars; §M7.MUSTNOT.3)
- `<out>/project.yaml` is generated by template override (`vendor/snakescale/project.yaml`
  as immutable template; only dynamic fields touched — see §M1.MUSTNOT.6)
- paired-end: R2 symlink created, but `input.fastq[GSM]` list holds R1 only
  (snakescale Snakefile:171,202 hardcodes `_1.fastq.gz`; see backlog)
